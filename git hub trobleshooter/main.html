Git Push Troubleshooter

Diagnose and fix issues with empty files after pushing to Git

Common Causes of Empty Files in Git

When files appear empty after pushing to Git, it's usually due to one of these reasons:

Potential Issues

  • Files were not properly added to staging
  • .gitignore rules are excluding your files
  • Line ending conflicts (CRLF vs LF)
  • File permission issues
  • Git hooks modifying content
  • Large File Storage (LFS) issues
  • Network issues during push

Diagnostic Steps

Check if files are actually in your repository:

git log --stat

Verify what's in your latest commit:

git show --name-only HEAD

Check if files are being ignored:

git check-ignore -v path/to/file

Step-by-Step Solution

1. Check Your Local Repository

First, verify that your files are actually committed locally:

# See status of files
git status

# See what's in the last commit
git show --name-only HEAD

# Compare with remote
git diff --name-only HEAD origin/main

2. Check .gitignore Rules

Your files might be excluded by .gitignore rules:

# Check if a file is ignored
git check-ignore -v path/to/file

# Check global gitignore
git config --get core.excludesfile

3. Re-add and Re-commit Files

If files were not properly added, try adding them again:

# Add specific file
git add path/to/file

# Or add all files
git add .

# Commit with a message
git commit -m "Re-adding missing files"

# Push to remote
git push origin main

4. Verify the Push Worked

After pushing, verify the files are on the remote:

# Check remote status
git remote show origin

# View files on remote
git ls-tree -r origin/main --name-only

Prevention Tips

Always Verify Your Changes

Before pushing, always check what you're about to push:

# See what will be pushed
git diff --stat --cached origin/main

# Or for a more detailed view
git diff --cached origin/main

Use Git Hooks for Validation

Create a pre-push hook to validate your changes:

# Create a pre-push hook
echo '#!/bin/sh

# Check for empty files
if git diff --cached --name-only | xargs -I {} sh -c \
"test -s {} || echo Error: {} is empty"; then
  echo "Aborting push due to empty files"
  exit 1
fi

exit 0' > .git/hooks/pre-push

chmod +x .git/hooks/pre-push

Regular Maintenance

Keep your Git repository healthy with regular maintenance:

# Verify repository integrity
git fsck

# Clean up unnecessary files
git gc --auto

# Prune remote tracking branches
git remote prune origin
